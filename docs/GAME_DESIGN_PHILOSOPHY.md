# FLASHEAD Game Design Philosophy

이 문서는 게임 디자인 관점의 핵심 철학을 정의합니다.
구현 세부는 `CODEMAP.md`를, AI 작업 규칙은 루트의 `../AGENTS.md`를,
비주얼 규칙은 `VISUAL_STYLE_GUIDELINES.md`를 참고하세요.

## 1. 인게임 통합 UI 철학

FLASHEAD는 "UI를 화면 위에 붙이는 게임"이 아니라,
"게임 오브젝트 자체가 정보를 말해주는 게임"을 지향합니다.

- 핵심 상태(생존/위협/목표)는 가능한 한 오브젝트에 통합해 표현합니다.
- 상단 고정 바/패널은 최소화하고, 필요할 때만 제한적으로 노출합니다.
- 플레이어가 시선을 게임 중심부에서 떼지 않아도 상태를 판단할 수 있어야 합니다.

### 현재 적용 원칙
- 플레이어 HP: 커서 통합형 HP 링으로 표현
- 보스 HP: 별도 상단 바가 아닌 인게임 통합형 표현을 유지
- `data/game-config.json`의 `hud.hpDisplay`는 레거시/예약 설정이며 상단 하트 UI로 사용하지 않음

## 2. 최소 UI 노출 원칙

UI는 많을수록 좋은 것이 아니라, 의사결정에 필요한 최소 정보만 즉시 보이면 됩니다.

- 상시 노출: 생존/판정에 즉시 필요한 정보
- 조건부 노출: 업그레이드 선택, 웨이브 전환 등 특정 상황에서만 필요한 정보
- 비상시 강조: 위험/보상 신호는 강하게, 짧게, 겹치지 않게 노출

### HUD 설계 기준
- HUD는 보조 정보(웨이브/시간/상호작용 상태) 중심으로 제한합니다.
- 핵심 전투 정보(HP/위협도)는 가능한 한 게임 오브젝트 내부 표현을 우선합니다.
- 새로운 상단 고정 UI 추가는 아래 조건을 모두 만족할 때만 허용합니다.
  - 오브젝트 통합 표현으로는 가독성을 충족할 수 없음
  - 플레이어 의사결정 속도에 직접 기여함
  - 화면 집중도를 해치지 않음

## 3. 피해 기믹 피드백 일관성

같은 유형의 기믹(폭탄, 레이저 등)은 발생 경로가 달라도 **동일한 피드백 언어**(색상/텍스트/이펙트)를 사용해야 합니다.

### 원칙
- 동일 기믹의 동일 결과(예: 폭탄 제거)는 표시 경로(일반 폭탄/낙하 폭탄)에 관계없이 같은 색상·텍스트를 사용
- 피해 기믹 추가 시 기존 동일 유형 기믹의 피드백 색상/텍스트를 먼저 확인하고 맞춤
- 피드백 색상은 결과의 **의미**(제거=CYAN, 폭발=빨강 등)에 따라 결정하며, 발생 경로에 따라 달라지지 않음

### 현재 적용 규칙
| 결과 | 색상 | 예시 |
|------|------|------|
| 피해 기믹 어빌리티 제거 | `COLORS.CYAN` | 폭탄/낙하폭탄 REMOVED |
| 피해 기믹 폭발 (피해 발생) | `0xff0044` | 폭탄/낙하폭탄 BOOM |

## 4. 구현 팀(개발자/AI) 적용 가이드

UI/게임 디자인 변경 작업 시 아래 순서를 따릅니다.

1. 먼저 "오브젝트 통합형 표현"으로 해결 가능한지 검토
2. 통합 표현이 가능하면 Renderer를 통해 구현
3. 별도 HUD가 필요하다면 근거(가독성/판정 속도)를 문서에 남기고 최소 노출로 설계
4. 비주얼 규칙(실루엣/형태/스타일)은 `VISUAL_STYLE_GUIDELINES.md`를 준수
5. 구조적 변화가 있다면 `CODEMAP.md`와 루트의 `../AGENTS.md`를 함께 업데이트
