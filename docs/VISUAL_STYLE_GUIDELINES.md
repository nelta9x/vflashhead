# FLASHEAD Visual Style Guidelines

이 문서는 게임의 시각적 규칙과 형태적 제약을 정의합니다.
게임 UX 철학은 `GAME_DESIGN_PHILOSOPHY.md`를 참고하세요.

## 1. 보스 HP 시각 표현 원칙

보스 HP는 별도 HUD 바가 아니라, 보스 외형(아머 실루엣) 자체로 표현합니다.

- 아머 실루엣 조각 수 자체가 HP 슬롯 수입니다.
- 슬롯 규칙: **100 HP당 1 슬롯, 나머지가 있으면 올림**
- 수식:
  - `slotCount = ceil(maxHp / 100)`
  - `filledSlots = ceil(currentHp / 100)` (HP가 0일 때 0)

### 예시
- `maxHp = 300` -> 슬롯 3개
- `maxHp = 301` -> 슬롯 4개
- `currentHp = 201 / 300` -> 채워진 슬롯 3개
- `currentHp = 200 / 300` -> 채워진 슬롯 2개

## 2. 형태(Shape) 제약

- 보스 외형은 원형/구형 실루엣 인상을 유지해야 합니다.
- 슬롯 수가 적어도 전체 외곽이 "부서진 조각 모음"처럼 보이지 않게,
  원형에 가깝게 조각 분포와 폭을 설계합니다.
- 직사각형 바 형태나 HUD형 체력바로 회귀하지 않습니다.

## 3. 렌더링 스타일 일관성

- 보스의 기존 아크/링 기반 비주얼 언어를 유지합니다.
- 실루엣 조각 변경은 색상/글로우/간격/폭을 함께 조정해 시각 균형을 유지합니다.
- "슬롯 규칙 충족"과 "원형 인상 유지"가 충돌하면,
  슬롯 규칙을 지키되 원형 인상을 최대한 보존하는 파라미터를 찾습니다.

## 4. 구현 가이드 (Renderer 중심)

- 형태 로직(슬롯 수 계산)은 로직 레이어에서 처리하고,
  실제 드로잉은 Renderer(`src/effects/*Renderer.ts`)에서 처리합니다.
- 보스 비주얼 변경 시 최소 확인 항목:
  - 슬롯 수 계산 정확성 (`100 HP = 1 슬롯`, 올림)
  - 원형 실루엣 유지 여부
  - 메인 메뉴 보스와 인게임 보스의 스타일 연속성
  - 별도 HUD 체력바 미도입 여부
